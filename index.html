part1_4_表达式_day1

基础
左值和右值：右值用的是对象的值，左值用的是对象的身份。
赋值运算使用左值做左对象，得到的仍然是一个左值。
取地址符作用于一个左值，返回一个右值。
decltype（）中的函数表达式求值结果如果是左值，则得到一个引用类型。
<<没有明确定义什么时候以及如何对运算对象求值，所以计算对象之间不要有关系。如cout<<i<<i++;是未定义的。
f()+g()*h();如果影响同一对象，则会产生未定义的行为。因为没有规定f(),g(),h()什么时候计算。
算数运算符
%取模，取余运算，m%n结果与同符号。
如果对象非常大，在范围for语句中使用引用来避免对对象的复制。
不要使用连续的关系运算符进行比较。例如：a<b<c;如果a<b,那么当c>1时表达式为真。
if(val==true)；只有当val=1时才为真。
赋值运算符：
赋值运算符的左侧必须是可修改的左值。
赋值运算满足右结合律。
赋值运算符的优先级较低，所以在条件语句中，建议加上括号。
递增和递减运算符：
前置版本：++i；后置版本：i++；除非必须，否则一律使用前置版本。后置版本会额外消耗资源。
后置版本在自身所在的表达式中，先将自身加1，在用原来没有加过的参加计算。
成员访问运算符：
(*p).size==p->size;
vector::iterator;//指向存有int类型元素的容器的迭代器。
条件运算符：
cond?expr1:expr2;cond值为真对ecpr1求值并返回。否则对expr2求值并返回。
位运算符：
建议只将位运算符用于处理无符号数。
<<,>>左移和右移运算符
~位求反运算符。
&，|，^位与，位或，位异或运算符。
sizeof运算符：
sizeof(type) 计算类型的大小
sizeof expr 计算变量实际类型的大小
Sales_data data,*p;
sizeof (Sales_data)==sizeof data ==sizeof *p;
sizeof不会实际计算运算对象的值。所以即使p未初始化也不会报错。
sizeof 会计算整个数组的大小。sizeof(ia)/sizeof(ia)返回数组元素的数量。string和vector不行。
类型转换：
显示转换：
cast-name(expreesion);
cast指定了只想的是哪一种转换。
static_cast:不包含底层const的转换，都可以使用static_cast；将较大的算术类型转化为较小的，如：将两个整型相除并转化为double。或者执行编译器无法自动执行的类型转换，如：将存入void中的指针恢复为原来的类型。
const_cast:只能用于改变运算对象的底层const。

part1_3_字符串向量数组_day1

命名空间的using说明
using namespace::name.
头文件不应包含using
标准库类型string
定义与初始化：string s1;
string s2=s1;
string s3=”hiya”;
string s4(“hiya”);
string s5(10,’c’);
使用等号是拷贝初始化，不使用等号是直接初始化
string操作，p77
读写string对象：
cin>>s;//遇到空格停止，如：“ abc de”，s=”abc”.
getline(cin,s);//遇到换行结束，会将换行符读入，但是不会将换行符写出。 如：“ abc de”，s=” abc de”.
while(cin>>s)cout<<s<<endl;//输入未知数量的string。
s.empty(),空则返回true，否则返回false
s.size()返回s的长度，返回的类型是string::size_type,这是一个无符号整型数，与之对比的数也会被转化为无符号数，例如：s.size()<-1,-1会被转化为一个很大的无符号数。
遇到s.size()时不要使用有符号数int与其计算。
字符串比较：==，!=,<,>……
string对象相加：两个字符串相加，得到一个新的字符串。
string和字面值相加：必须保证每一个＋的两边至少有一个string
string s=s1+s2+’aaa’;√
string s=’aaa’+’bbb’+s2;×
string s=’aaa’+s2+’bbb’;√从左到右依次相加
处理string对象中的字符
cctype头文件，p82
处理所有字符：范围for语句for(auto c:s)将s中的每一个字符依次转化为c。for(auto c,s)cout<<c;//打印s中的每一个字符。
改变字符串中的字符：将范围for语句中的循环变量定义为引用。for(auto &c,s)c=toupper(c);cout<<s;//将s中的小写字母以大写输出。
处理单个或多个字符：下标，s[0]~s[s.size()-1];
标准库类型vector（容器）
头文件 #include
vector是模板，用<>来指定vector实例化成为什么样的类型。注意：类型不能是引用。
定义和初始化vector类型对象：
vector v1/v1(v2)/v1=v2……
vectorv(unsigned x，T s);圆括号里为元素个数，s为初始化的元素，省略则默认初始化。
vectorv{Ts…};花括号为元素，写几个元素就包含几个元素。
对于string类型，vectorv{10,”cd”}==vectorv(10,”cd”);//表示有10个cd。
列表初始化：vectorv1{……};//只能使用花括号初始化。
向vector中添加元素：push_back成员函数，将目标元素压入vector类型的尾部。v.push_back(1);//将1压入v的尾部。
vector对象不建议定义大小。
vector操作：类似string，不能使用下标来添加元素。
为确保下标合法化建议尽可能使用范围for语句。
迭代器
auto b=v.begin(),c=v.end();//begin指向第一个元素，end指向最后一个元素的下一位置，成为尾迭代器。
如果容器为空，begin和end返回的是同一个迭代器，都是尾迭代器。
对容器使用for语句时需要使用！=而非<,>因为容器元素并非支持<,>运算。
迭代器的运算符：*iter；返回迭代器所指元素的引用。
++iter，–tier。
c++11引入cbegin和cend专门获得常量类型的迭代器。
结合解引用：(*it).empty()==it->empty()，it所指的对象是否为空。
向一个容器中添加了元素，会使其迭代器失效。
迭代器运算：+，-，>,<……

数组
数组大小确定不变。
定义与初始化：数组的维度必须是常量表达式。int a[b];b必须是const unsigned而不能是unsigned
不允许使用引用的数组。int&a[5];×
显示初始化：int a[3]={1,2,3};
int a[]={1,2,3};
字符数组：可以用上述初始化方式和int c[]=”hello”;//c的长度为字符串长度加1，最后一格会加上一个空格。
不允许使用一个数组初始化或者赋值给另一个数组。
允许定义数组的指针以及数组的引用：int(*array)[10]=&arr;int(&array)[10]=arr;//从括号内向外，再从左向右。
在函数内部定义的数组默认初始化会给与不确定的值。
数组与指针：int p=nums;==int p &nums[0];
一些情况下数组的操作是指针的操作。int ia[]={……};auto a2(ia); a2是指向ia首地址的指针。
但是decltype(ia) a3={……};是正确的，decltype返回的是数组类型。
指针也是迭代器：指针适用vector和string迭代器所支持的所有运算。
a[10]数组。可以用int p=a[10];来获得指向a数组尾元素下以位置的指针。类似于end迭代器。
c++11引入begin和end函数来获取数组的头尾指针。头文件iterator
int first=begin(a);int last=end(a);
下标和指针：只要指针指向数组中的元素，指针可以进行下标运算。intp=&a[2];cout<<*(p+2);
利用数组初始化容器：int a[]={1,2,3,4,5};vectorv(begin(a),end(a));
多维数组
c++没有多维数组，只有数组的数组。
int a[3][4];
多维数组与指针：数组a中包含了3个元素，每个元素是拥有4个元素的数组。
所以 数组a的指针是：int (*p)[4]=a/begin(a);//p是指向包含4个整型数组的指针。
多维数组的初始化：int[3][[4]={ {1,2,3，3},{4,5,6，6},{7,8,9，9} };内层花括号非必须。
使用范围for语句时，如果在最内层之外不使用引用，auto得到的类型会是指针，程序会无法编译。

part1_2_变量和基本类型_day2

指针
定义 type *name，name为指向type类型的指针
指针存放某个对象的地址
不能定义指向引用的指针
指针类型必须和其所指对象一致（例外1：允许一个常量的指针指向一个非常量对象）
*、&在声明语句中用于组成符合类型int *p是一个指针，int &p是一个引用
在表达式中为运算符，a=p表示的是将指针p指向的内容赋值给a，p=&a表示的是将a的地址赋值给指针p
空指针：int *p=nullptr(c++11)推荐使用
int *p=0
int *p=NULL
int a=0;int p=a;(×)
建议初始化所有指针
void 空指针可以存放任意对象的地址
指向指针的指针 int *q; int**p=&q;p为指向指针的指针
指向指针的引用 int *q; int &p=q;p为指向指针的引用
对声明语句从右向左读有助于理解变量
对于指针在表达式中 p代表指向的对象，p代表指针地址
对于普通变量在表达式中 p代表值，&p代表地址。
const限定符
定义一个值不能被改变的变量
const int bufsize=100；
const定义的常量只在当前文件中有效
extern const int bufsize=100;
次变量可以在不同文件中有效
const的引用：
只能用常量引用来给常量取别名
const int a=100;
const int &b=a;
int &c=a(×)
const引用可能引用的是一个并非常量的对象
int a = 1;
int& b = a;
const int & c = a;
std::cout << c;//输出1
b = 2;
std::cout << c;//输出2
允许一个常量的指针指向一个非常量对象
const指针：
const name=&obj；
指向常量的指针才能指向常量，并且不能改变指向对象的值。
int b=10;int const a=&b;a为指向int类型的常量指针
const double c=10;const double const d=&c;d为指向双精度浮点数常量的常量指针（运用声明语句从右到左读）
注意：const int &a=0；是对的。
int &a=0；是错的。
顶层const：https://blog.csdn.net/qq_19528953/article/details/50922303
const修饰的对象不能被修改。
常量表达式：
值不会改变并且在编译过程就能得到结果的表达式。
声明：c++11规定可以用constexpr来定义一个常量表达式。
constexpr size=0;
constexpr int p=&c;p是一个顶层const；p是一个指向整数的常量指针。（与const int p有很大区别)
处理类型
类型别名：typedef double wage,p;wage是double的别名，p是double的别名。
c++11：using wage=double；
auto类型：必须初始化。auto类型的变量会根据初始化的结果推断变量的类型。auto会忽略掉顶层const，保留底层const
const int a=i;&b=a;
auto s1=a/b;s1为int
auto p=&a;p是指向int类型常量的指针（对常量取地址是一种底层const）
decltype类型指示符：decltype(f()) sum=x;sum类型为f的返回类型。=x为初始化。
decltype（p）得到的是引用，p被当作表达式，解引用会得到引用类型
decltype（（i））得到的是引用，变量加上括号，系统会认为它是表达式，并得到引用类型
decltype（a=b）赋值语句会产生引用类型
引用类型必须初始化。
自定义数据结构：
struct name{}a,*p;
c++11规定可以给类内成员定义一个类内初始值。
头文件保护#define

part1_2_变量和基本类型_day1

算术类型
带符号类型int short、long、long long。可以在他们前面添加unsigned以得到无符号类型，unsigned int可以简写为unsigned
char类型和bool类型不要出现在算数表达式中，char类型在有些机器上有符号，而在另外一些机器上没有。
浮点数运算推荐使用double而不是float
当赋值给无符号类型超出它表示范围的值时，实际得到的值是初始值对无符号类型表示数值总值取模后的余数
当赋值给带符号类型超出它表示范围的值时，结果是未定义的
当一个 算术表达式中既有无符号数又有有符号数时，有符号数会自动转化为无符号数
变量
初始化：初始化！=赋值
列表初始化：int a=0;int a={0};int a{0};int a(0);2,3为列表初始化
当列表初始化涉及到可能丢失信息时，初始化不会被执行
如int a{1.1}是错误的
默认初始化：没有显式初始化的变量会被赋予默认值。但是定义在函数体内部的变量不会被默认初始化。类的对象有类决定。
变量声明和定义：声明使得变量为程序所知，声明了变量即可使用在其他程序定义的名字
声明方法：在变量名前添加extern并且不要显式地初始化变量。
引用
指的是左值引用，即为变量取一个新的名字
int a=0;int &b=a;b指向了a
